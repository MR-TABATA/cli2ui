What's in the box

A real DB-client layout, one panel per job.

SQL runner

Read-only ad-hoc queries by default (READ ONLY + statement timeout + 1000-row cap), with an opt-in write mode that commits — guarded by a whole-database safety snapshot before each write.

EXPLAIN snapshots + diff

Save query plans and diff two of them — before/after an index — instead of copy-pasting plans into a scratch file.

Activity

Running queries + connections from pg_stat_activity, with one-click cancel / kill.

Locks

Sessions blocked on a lock paired with the holder (pg_blocking_pids), plus one-click cancel / kill of the blocker.

Backup / restore

Automatic table snapshots before every destructive or structural change, plus restore of an uploaded dump — streamed to the client tool, not buffered in memory.

Replication

Readiness check (wal_level / max_wal_senders) + WAL position, connected standbys, and slot create / drop.

Health · bloat estimate

Wasted table space from a stats-only query (no scan), next to the dead-rows / vacuum card.

Objects browser

Databases, schemas and roles (\l \dn \du) — read-only — plus a postgresql.conf editor with reload-vs-restart badges.

Command history

Every SQL run logged with status, row count and timing — and one-click re-open to run it again.

Not doing — on purpose

The constraints are the product.

⊘ No AI

API-key management plus sending your schema to a third party is a non-starter for the people this is for. The tool does exactly the command you clicked — nothing it guessed.

⊘ No SaaS

The moment we'd hold your DB connection info, the liability and encryption story swamps the project. Local-only keeps it honest — no DB creds ever leave your network.
